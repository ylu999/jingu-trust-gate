Jingu Harness is a governed acceptance layer for LLM-powered systems.

Jingu ensures LLM outputs are verified, constrained, and accepted before they affect your system.

Without vs With Jingu

Without Jingu

AI output goes directly into your system.

  • Silent bugs reach production
  • Invalid JSON stored in your database
  • Incorrect state mutations applied

With Jingu

Every output is verified before being accepted.

  • Only tested changes are applied
  • Structure and schema validated
  • Controlled, auditable state transitions

How it works

LLM / Agent Output | Jingu Harness +-- Verify (tests, schema, semantic) +-- Invariants (scope, integrity, regression) | Accept / Reject / Retry | System State

On failure, Jingu builds structured feedback and passes it back to the agent. The loop continues until the output is accepted — or escalated.

Demo — AI fixes a bug

The agent tries three times. Jingu governs each attempt:

Iteration 1 -> VERIFY_FAIL // test failed: add(1,2) returned -1 | strategy: retry — feedback sent to agent Iteration 2 -> REGRESSION // multiply broke: 6 -> 0.666 | strategy: rollback_and_retry — agent must fix without breaking tests Iteration 3 -> ACCEPTED // all tests pass, no regressions
